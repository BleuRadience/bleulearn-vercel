DOCUMENTED LESSON PLAN — Grades K-12

Accessing a Public Archive API to Find Historical Documents

Documented Learning Objective

By the end of this lesson, scholars will be able to write a Python script that queries the Chronicling America API and documents debugging steps in a Code Journal.

Free Resources (No Textbook)

Chronicling America API (chroniclingamerica.loc.gov/about/api); Replit (replit.com, free, browser-based)

Vocabulary (Frayer Model Pre-Teach)

  • API: a defined way for one program to request data from another
  • JSON: a structured text format for exchanging data

Instruction — Steps

  1. Write pseudocode before any code: import library, build query, GET request, parse, print results.
  2. Write the actual request and run it.
  3. Debug the inevitable KeyError by printing the actual response structure.
  4. Verify by checking that returned URLs resolve to real Library of Congress pages.

Discourse

Partner Discussion: what did debugging the KeyError teach you about not trusting documentation blindly?

BleuLearn Pause

"What was the gap between what you expected the code to do and what it actually did?"

Exit Ticket

"Document one specific bug you fixed today and exactly how you found it."

Differentiation

  • Bridge: starter code provided with one intentional bug to find.
  • Grade-Level: build the full script independently.
  • Gifted: filter results to a specific publication and add full-text download.

Archive Assignment

Use your working script to find one documented article relevant to a current BleuHistory unit; cite it properly.
